3 Abordagens para a nova estrutura

Como organizar sidebar + páginas no Orderly Checkout V2

A — Sidebar de ícones (icon-only)

Sidebar estreita (56px) sempre visível com ícones. Simples e discreta, não rouba espaço do conteúdo.

Prós

  • Não interfere no layout atual
  • Rápido de implementar

Contras

  • Sem labels visíveis nos links
Orderly v2
🏠 Dashboard
📋 Logs de API
✅ Backlog
⚠️ Pendências

B — Sidebar expandida com labels ⭐ Recomendado

Sidebar larga (200px) com ícone + texto para cada página. Layout AppShell compartilhado entre todas as rotas.

Prós

  • Navegação clara com labels
  • Arquitetura limpa com AppShell
  • Escalável para novas páginas

Contras

  • Ocupa 200px horizontais
Orderly v2
🏠 Dashboard
📋 Logs de API
✅ Backlog
⚠️ Pendências
sidebar colapsada → ícones

C — Sidebar colapsável (expandida/ícones)

Sidebar que alterna entre modo expandido (200px) e só ícones (56px) via botão de toggle.

Prós

  • Flexível para o usuário
  • Melhor em telas pequenas

Contras

  • Mais complexo de implementar
  • Estado extra para gerenciar
